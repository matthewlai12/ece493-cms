# Acceptance Test Suite — UC-15 Access Assigned Papers

## Overview
**Use Case:** UC-15 Access Assigned Papers  
**Primary Actor:** Referee  
**Objective:** Verify an authenticated referee can view their assigned papers list and open manuscript files from storage. Verify behavior for no assignments, file issues, DB/server failures, and canceling access.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- Referee must be authenticated to access assigned papers.
- Assigned papers appear only after the referee accepts invitations (UC-14 accept path).
- Manuscripts are stored in a file storage system and accessed via the CMS.
- The manuscript may open in-browser (PDF viewer) or download, depending on implementation.

**R-01 Referee (has assigned papers)**
- Email/Username: `referee_user@example.com`
- Password: `ValidPass!234`
- Role: Referee

**R-02 Referee (no assigned papers)**
- Email/Username: `new_referee@example.com`
- Password: `ValidPass!234`
- Role: Referee

### Paper / Assignment Data
**P-01 Assigned paper (healthy file)**
- Paper ID: `PAPER-301`
- Assigned to: R-01
- Manuscript file: available in storage (e.g., `paper_301.pdf`)

**P-02 Assigned paper (file unavailable/corrupted)**
- Paper ID: `PAPER-302`
- Assigned to: R-01
- Manuscript file: missing/corrupted/unreadable in storage

## UC-15-AT-01 — Assigned Papers option is available after login
**Priority:** P0  
**Preconditions:** Referee is authenticated (R-01).  
**Steps:**
1. Log in as referee (R-01).
2. Navigate to the dashboard.
3. Locate “Assigned Papers” (or equivalent).
**Expected Results:**
- “Assigned Papers” option is visible and accessible.

---

## UC-15-AT-02 — Referee can view list of assigned papers (Main Success Path)
**Priority:** P0  
**Test Data:** R-01 (assigned P-01)  
**Preconditions:** R-01 has at least one assigned paper in DB.  
**Steps:**
1. Log in as R-01.
2. Go to Assigned Papers.
**Expected Results:**
- CMS retrieves assigned papers list from the database.
- CMS displays the list of assigned papers.
- List entries show identifying info (e.g., title, paper ID, status) as available.

---

## UC-15-AT-03 — Referee can open an assigned paper manuscript (Main Success Path)
**Priority:** P0  
**Test Data:** R-01 + P-01  
**Preconditions:** P-01 manuscript exists in storage and is linked to the paper.  
**Steps:**
1. From assigned papers list, select P-01.
2. Choose “Open/View/Download Manuscript”.
**Expected Results:**
- CMS retrieves the manuscript file from file storage.
- Manuscript is opened in-browser or downloaded successfully.
- The file content corresponds to P-01.

---

## UC-15-AT-04 — No assigned papers shows empty-state message (Extension 4a)
**Priority:** P0  
**Test Data:** R-02  
**Preconditions:** R-02 has no assigned papers.  
**Steps:**
1. Log in as referee (R-02).
2. Go to Assigned Papers.
**Expected Results:**
- CMS displays a message indicating no papers are assigned.
- No errors are shown.

---

## UC-15-AT-05 — Manuscript unavailable/corrupted shows friendly error (Extension 7a)
**Priority:** P0  
**Test Data:** R-01 + P-02  
**Preconditions:** P-02 is assigned to R-01; file is missing/corrupt in storage.  
**Steps:**
1. Log in as R-01.
2. Go to Assigned Papers.
3. Select P-02 and attempt to open the manuscript.
**Expected Results:**
- CMS displays an error indicating the paper cannot be accessed at this time.
- No sensitive technical details exposed (no stack trace, storage keys, internal paths).
- Referee remains able to return to assigned paper list.

---

## UC-15-AT-06 — Database unavailable prevents retrieving assigned papers list (Extension 4b)
**Priority:** P0  
**Test Data:** R-01  
**Preconditions:** Simulate DB outage/unavailability at the time list is loaded.  
**Steps:**
1. Log in as R-01.
2. Navigate to Assigned Papers while DB is unavailable.
**Expected Results:**
- CMS displays an error indicating assigned papers cannot be retrieved.
- No sensitive technical details are exposed.
- Page does not show incorrect/partial list data as if successful.

---

## UC-15-AT-07 — Referee cancels paper access (Extension 6a)
**Priority:** P2  
**Preconditions:** A cancel/back action exists when opening a manuscript (e.g., confirmation modal, download prompt, or in-viewer back).  
**Steps:**
1. Log in as R-01.
2. Navigate to Assigned Papers list.
3. Select P-01.
4. Initiate open/download.
5. Cancel the action (cancel dialog/download prompt or use “Back” before load completes).
**Expected Results:**
- CMS returns referee to assigned papers list (or remains on details page) without opening/downloading the file.
- No state change occurs (assignment remains assigned).

---

## UC-15-AT-08 — Referee can only access papers assigned to them (authorization check)
**Priority:** P0  
**Preconditions:** Another referee exists with a different assignment; or P-01 is *not* assigned to R-02.  
**Steps:**
1. Log in as R-02.
2. Attempt to access P-01 via direct URL (if known) or by manipulating request (e.g., paper ID in URL).
**Expected Results:**
- CMS denies access (403/Not authorized) or redirects safely.
- Manuscript is not exposed to unauthorized referee.

---

## UC-15-AT-09 — Web server unavailable behavior (Extension 1a)
**Priority:** P0  
**Preconditions:** Simulate CMS web server down/unreachable.  
**Steps:**
1. Attempt to access Assigned Papers page.
2. Attempt to open a manuscript.
**Expected Results:**
- User receives unavailability response (browser error or CMS error page).
- If CMS error page is shown, it indicates temporary service unavailability.
- No access occurs.

---

## UC-15-AT-10 — Cross-browser access/open behavior is consistent
**Priority:** P2  
**Test Data:** R-01 + P-01  
**Preconditions:** None.  
**Steps:**
1. Open assigned papers list and open manuscript in Chrome.
2. Repeat in Firefox.
**Expected Results:**
- List display and manuscript open/download behavior is consistent across browsers.
