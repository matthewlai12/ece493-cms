# Acceptance Test Suite — UC-18 Assign Referees to Submitted Papers

## Overview
**Use Case:** UC-18 Assign Referees to Submitted Papers  
**Primary Actor:** Editor  
**Objective:** Verify an authenticated editor can assign one or more referees to a submitted paper, that workload and per-paper constraints are enforced, assignments are stored, invitations are sent, and failure scenarios are handled correctly.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- Editor must be authenticated to assign referees.
- Reviewer workload limit exists (max assigned papers per referee = 5).
- Maximum referees per paper exists (e.g., 3 reviewers per paper).
- Assigning a referee creates a **pending invitation** (or equivalent) and triggers notification.
- A list of eligible referees is available in the CMS.

### Seed Accounts / Entities
**E-01 Editor**
- Email/Username: `editor_user@example.com`
- Password: `ValidPass!234`
- Role: Editor

**R-01 Referee (eligible)**
- Email: `referee_ok@example.com`
- Assigned papers: 0–4

**R-02 Referee (at workload limit)**
- Email: `referee_full@example.com`
- Assigned papers: 5

**R-03 Invalid/unavailable referee**
- Email: `invalid_referee@example.com`
- Not present in referee list / not registered

### Paper Data
**P-01 Submitted paper (needs reviewers)**
- Paper ID: `PAPER-601`
- Status: Submitted
- Current assigned reviewers: 0

**P-02 Submitted paper (already at max reviewers)**
- Paper ID: `PAPER-602`
- Status: Submitted
- Current assigned reviewers: max (e.g., 3)

## UC-18-AT-01 — Referee assignment interface is available for submitted papers
**Priority:** P0  
**Preconditions:** Editor is authenticated (E-01); P-01 exists with status Submitted.  
**Steps:**
1. Log in as editor (E-01).
2. Navigate to “Submitted Papers”.
3. Select P-01.
4. Open the referee assignment interface.
**Expected Results:**
- CMS displays referee assignment UI.
- CMS shows available referees list (or selection control).

---

## UC-18-AT-02 — Assign one referee successfully (Main Success Path)
**Priority:** P0  
**Test Data:** E-01 + P-01 + R-01  
**Preconditions:** R-01 is eligible (below workload max).  
**Steps:**
1. In assignment UI for P-01, select R-01.
2. Confirm assignment.
**Expected Results:**
- CMS checks R-01 workload (≤ 5).
- CMS assigns R-01 to P-01.
- Assignment is recorded in DB.
- Pending invitation is created for R-01.
- Notification to R-01 is triggered.
- CMS confirms to editor that assignment succeeded.

---

## UC-18-AT-03 — Assign multiple referees successfully
**Priority:** P1  
**Test Data:** E-01 + P-01 + (R-01 plus additional eligible referees)  
**Preconditions:** P-01 has fewer than max reviewers. All selected referees are eligible.  
**Steps:**
1. Select 2–3 eligible referees in the assignment UI.
2. Confirm assignment.
**Expected Results:**
- CMS validates workload for each selected referee.
- CMS records each assignment.
- CMS creates pending invitations for each referee.
- CMS triggers notifications to each referee.
- CMS confirms all assignments.

---

## UC-18-AT-04 — Workload limit blocks selecting an overloaded referee (Extension 6a)
**Priority:** P0  
**Test Data:** E-01 + P-01 + R-02  
**Preconditions:** R-02 is at workload limit (5).  
**Steps:**
1. Attempt to select/assign R-02 to P-01.
**Expected Results:**
- CMS prevents assignment of R-02.
- CMS informs editor that referee has reached maximum workload.
- No assignment record is created for R-02.
- No invitation/notification is created for R-02.

---

## UC-18-AT-05 — Paper reviewer limit blocks further assignments (Extension 6b)
**Priority:** P0  
**Test Data:** E-01 + P-02 + R-01  
**Preconditions:** P-02 already has max reviewers (e.g., 3).  
**Steps:**
1. Open assignment UI for P-02.
2. Attempt to assign an additional referee.
**Expected Results:**
- CMS blocks assignment.
- CMS informs editor that the paper already has the maximum allowed reviewers.
- No new assignment record is created.
- No invitation/notification is generated.

---

## UC-18-AT-06 — Invalid/unavailable referee selection is rejected (Extension 5a)
**Priority:** P1  
**Test Data:** E-01 + P-01 + R-03  
**Preconditions:** R-03 is not in referee list / not registered.  
**Steps:**
1. Attempt to assign an invalid or unavailable referee (e.g., via manual email entry or stale UI entry).
**Expected Results:**
- CMS displays an error indicating invalid referee selection.
- CMS requests valid selection.
- No assignment record is created.
- No invitation/notification is created.

---

## UC-18-AT-07 — Database unavailable prevents saving assignments (Extension 8a)
**Priority:** P0  
**Test Data:** E-01 + P-01 + R-01  
**Preconditions:** Simulate DB outage at the time of assignment confirmation.  
**Steps:**
1. Attempt to assign R-01 to P-01 while DB is unavailable.
**Expected Results:**
- CMS displays an error indicating assignments could not be saved.
- No assignment is persisted.
- No invitation/notification is sent (since save failed).

---

## UC-18-AT-08 — Invitations are visible to referees after assignment
**Priority:** P1  
**Test Data:** R-01 + assignment from AT-02  
**Preconditions:** Pending invitation created for R-01.  
**Steps:**
1. Log in as referee (R-01).
2. Navigate to “Review Invitations”.
**Expected Results:**
- The invitation for P-01 appears as Pending.
- Referee can open invitation to view details and respond (UC-14).

---

## UC-18-AT-09 — Web server unavailable behavior (Extension 1a)
**Priority:** P0  
**Preconditions:** Simulate CMS web server down/unreachable.  
**Steps:**
1. Attempt to access submitted papers list and assign referees.
**Expected Results:**
- User receives unavailability response (browser error or CMS error page).
- No assignment occurs.

---

## UC-18-AT-10 — Security: non-editor cannot assign referees
**Priority:** P0  
**Preconditions:** A non-editor account exists (e.g., referee or author).  
**Steps:**
1. Log in as a non-editor user.
2. Attempt to access the assignment interface (via UI or direct URL).
**Expected Results:**
- CMS denies access (403/Not authorized) or redirects safely.
- No assignment can be performed.

---

## UC-18-AT-11 — Cross-browser assignment behavior is consistent
**Priority:** P2  
**Test Data:** E-01 + P-01 + R-01  
**Preconditions:** None.  
**Steps:**
1. Assign a referee in Chrome (use fresh paper or reset state).
2. Assign a referee in Firefox (use another paper or reset state).
**Expected Results:**
- Workload checks, saving, and confirmations behave consistently across browsers.
