# Use Case UC-04: Register User Account

**Goal in Context**:  
Allow a new user to create a CMS account using a unique and valid email address in order to access system features.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Registered User (new user)

**Secondary Actors**:  
CMS Web Interface  
CMS Database  
Email Validation Service

**Trigger**:  
The user selects the “Register” option on the CMS website.

---

## Success End Condition
* A new user account is successfully created and stored in the CMS database, and the user is redirected to the login page.

## Failed End Condition
* The user account is not created, and the user remains unregistered.

---

## Preconditions
* The CMS web application is accessible via a supported web browser.
* The user does not already have an account with the same email address.

---

## Main Success Scenario
1. The user accesses the CMS homepage.
2. The CMS displays an option to register a new account.
3. The user selects the register option.
4. The CMS displays the registration form.
5. The user enters required registration information, including email and password.
6. The CMS validates the email format and password security requirements.
7. The CMS verifies that the email address is unique.
8. The CMS stores the new user information in the database.
9. The CMS redirects the user to the login page.

---

## Extensions

* **6a**: Email format is invalid  
    * **6a1**: The CMS displays an error message indicating the email format is invalid and requests correction.

* **6b**: Password does not meet security standards  
    * **6b1**: The CMS displays an error message indicating password requirements are not met.

* **7a**: Email address already exists in the system  
    * **7a1**: The CMS displays an error message stating the email is already registered.

* **8a**: Database is unavailable  
    * **8a1**: The CMS displays an error message indicating account creation failed and does not store user information.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent (new users register for each conference cycle)  
* **Open Issues**:  
  * Whether email verification is required before first login  
  * Whether CAPTCHA should be used to prevent automated registrations
