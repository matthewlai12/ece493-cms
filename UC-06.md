# Use Case UC-06: Log In to System

**Goal in Context**:  
Allow a registered user to authenticate using valid credentials in order to access their personalized dashboard and system features.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Registered User

**Secondary Actors**:  
CMS Web Interface  
CMS Database  
Authentication Service

**Trigger**:  
The user selects the “Log In” option on the CMS website.

---

## Success End Condition
* The user is successfully authenticated and redirected to their personalized dashboard.

## Failed End Condition
* The user is not authenticated and cannot access protected CMS features.

---

## Preconditions
* The user has previously registered an account in the CMS.
* The CMS web application is accessible via a supported web browser.

---

## Main Success Scenario
1. The user accesses the CMS homepage.
2. The CMS displays an option to log in.
3. The user selects the log-in option.
4. The CMS displays the login form.
5. The user enters their email/username and password.
6. The CMS validates the submitted credentials against stored user records.
7. The CMS authenticates the user.
8. The CMS redirects the user to their personalized dashboard.

---

## Extensions

* **5a**: Username or password is incorrect  
    * **5a1**: The CMS displays an error message indicating invalid credentials and allows the user to retry.

* **6a**: User account does not exist  
    * **6a1**: The CMS displays an error message indicating the account was not found and suggests registration.

* **6b**: User account is locked or disabled  
    * **6b1**: The CMS displays a message indicating the account is unavailable and advises contacting system support.

* **6c**: CMS database is unavailable  
    * **6c1**: The CMS displays an error message indicating authentication cannot be completed at this time.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Very frequent (performed by all registered users)  
* **Open Issues**:  
  * Whether multi-factor authentication should be supported  
  * Whether repeated failed login attempts should trigger account lockout
