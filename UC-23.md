# Use Case UC-23: Generate Conference Schedule

**Goal in Context**:  
Automatically generate a complete conference schedule that assigns accepted papers to sessions, times, and rooms in an efficient and conflict-free manner.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Administrator

**Secondary Actors**:  
Editor  
CMS Scheduling Engine  
CMS Web Interface  
CMS Database

**Trigger**:  
The administrator requests automatic generation of the conference schedule.

---

## Success End Condition
* A valid conference schedule is generated, stored, and made available for review and further editing.

## Failed End Condition
* The conference schedule is not generated due to errors, missing data, or violated constraints.

---

## Preconditions
* The administrator is registered and authenticated in the CMS.
* A set of accepted papers exists.
* Required scheduling data (rooms, time slots, constraints) is available.
* The CMS scheduling algorithm is available.

---

## Main Success Scenario
1. The administrator logs into the CMS.
2. The administrator navigates to the schedule management area.
3. The administrator selects the option to generate the conference schedule.
4. The CMS retrieves accepted papers and scheduling constraints from the database.
5. The CMS executes the scheduling algorithm.
6. The CMS generates a conference schedule that assigns papers to sessions, rooms, and time slots.
7. The CMS stores the generated schedule in the database.
8. The CMS displays the generated schedule to the administrator for review.

---

## Extensions

* **4a**: Required scheduling data is missing or incomplete  
    * **4a1**: The CMS displays an error message indicating missing data.
    * **4a2**: The CMS aborts schedule generation.

* **5a**: Scheduling algorithm fails to find a valid schedule  
    * **5a1**: The CMS displays an error message indicating that no valid schedule could be generated.
    * **5a2**: The CMS does not store a schedule.

* **7a**: CMS database is unavailable  
    * **7a1**: The CMS displays an error message indicating that the schedule could not be saved.

* **3a**: Administrator cancels schedule generation  
    * **3a1**: The CMS stops the generation process and does not store a schedule.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Typically once per conference, with possible re-generation  
* **Open Issues**:  
  * Whether administrators can configure or tune scheduling constraints before generation  
  * Whether partial schedules should be generated if constraints cannot all be satisfied
