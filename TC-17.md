# Acceptance Test Suite — UC-17 Enforce Reviewer Workload Limit

## Overview
**Use Case:** UC-17 Enforce Reviewer Workload Limit  
**Primary Actor:** Referee  
**Secondary Actors:** Editor  
**Objective:** Verify that the CMS enforces a strict maximum of five review assignments per referee, blocking any editor assignment or referee acceptance that would exceed this limit, and handling error conditions correctly.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- Maximum review limit = **5 papers per referee**.
- Both **editor-driven assignment** and **referee-driven invitation acceptance** must enforce the same limit.
- The CMS maintains an accurate, real-time count of assigned papers per referee.

**R-01 Referee (below limit)**

**R-02 Referee (at limit)**
- Email/Username: `referee_full@example.com`

- Assigned papers: 5

**E-01 Editor**
- Email/Username: `editor_user@example.com`

## UC-17-AT-01 — Assignment allowed when referee is below workload limit (Main Success Path)
**Priority:** P0  
**Test Data:** E-01 + R-01 + P-01  
**Preconditions:** R-01 has 4 assigned papers (below limit).  
**Steps:**
1. Log in as editor (E-01).
2. Select paper P-01.
3. Assign referee R-01.
**Expected Results:**
- CMS retrieves R-01’s current assignment count.
- CMS determines assignment will not exceed limit.
- Assignment is allowed.
- R-01’s assigned paper count becomes 5.
- CMS confirms assignment success.

---

## UC-17-AT-02 — Referee acceptance allowed when below workload limit (Main Success Path)
**Priority:** P0  
**Test Data:** R-01 + INV-01  
**Preconditions:** R-01 has 4 assigned papers; INV-01 is pending for R-01.  
**Steps:**
1. Log in as referee (R-01).
2. Open pending invitation INV-01.
3. Click “Accept”.
**Expected Results:**
- CMS checks current assignment count.
- CMS allows acceptance.
- Invitation status becomes Accepted.
- Assigned paper count updates to 5.
- CMS confirms acceptance.

---

## UC-17-AT-03 — Editor assignment blocked when referee is at limit (Extension 3a)
**Priority:** P0  
**Test Data:** E-01 + R-02 + P-01  
**Preconditions:** R-02 already has 5 assigned papers.  
**Steps:**
1. Log in as editor (E-01).
2. Attempt to assign R-02 to P-01.
**Expected Results:**
- CMS detects assignment would exceed limit.
- CMS blocks the assignment.
- CMS displays a message indicating referee has reached the maximum review limit.
- CMS informs editor that the referee cannot be assigned additional papers.
- No assignment is recorded.

---

## UC-17-AT-04 — Referee acceptance blocked when at workload limit (Extension 3a)
**Priority:** P0  
**Test Data:** R-02 + INV-01  
**Preconditions:** R-02 has 5 assigned papers; INV-01 pending for R-02.  
**Steps:**
1. Log in as referee (R-02).
2. Open INV-01.
3. Click “Accept”.
**Expected Results:**
- CMS blocks acceptance.
- CMS displays a message indicating the maximum review limit has been reached.
- Invitation remains pending (or unresolved).
- Assigned paper count remains unchanged.

## UC-17-AT-05 — Workload limit enforcement is consistent across assignment paths
**Priority:** P1  
**Preconditions:** Same referee tested via editor assignment and referee acceptance.  
**Steps:**
1. Attempt editor assignment when referee is at limit.
2. Attempt referee acceptance when referee is at limit.
**Expected Results:**
- Both paths enforce the same limit.
- No path allows exceeding 5 assignments.

## UC-17-AT-06 — Database unavailable prevents workload verification (Extension 2a)

- No partial assignment is recorded.

## UC-17-AT-08 — Workload count updates immediately after assignment
**Priority:** P1  
**Preconditions:** R-01 has 4 assigned papers.  
**Steps:**
1. Assign R-01 to a fifth paper.
2. Immediately attempt to assign R-01 to another paper.
**Expected Results:**
- First assignment succeeds.
- Second assignment is blocked due to updated workload count.
- No race condition allows exceeding the limit.
