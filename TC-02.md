# Acceptance Test Suite — UC-02 View Conference Registration Prices

## Overview
**Use Case:** UC-02 View Conference Registration Prices  
**Primary Actor:** Public User (Guest)  
**Objective:** Verify guests can view the conference registration price list without authentication, and all alternative paths behave correctly.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- Base URL: `https://<cms-host>/`
- Pricing is stored in the CMS database and displayed on a Pricing/Registration Prices page/section.
- The pricing list contains prices by **attendance type** (e.g., Student / Regular / VIP).

### Seed Data Sets
**DS-01 (Normal pricing set)**
- Regular: $200
- Student: $100
- Industry: $250

**DS-02 (Pricing not configured)**
- No pricing records exist OR pricing feature toggled off/unpublished.

---

## UC-02-AT-01 — Guest can access homepage and see pricing navigation
**Priority:** P0  
**Preconditions:** CMS deployed and reachable.  
**Steps:**
1. Open the CMS base URL in a supported browser.
**Expected Results:**
- Homepage loads successfully (HTTP 200).
- No login is required to view the homepage.
- Navigation includes a link to “Pricing”, “Registration Prices”, or “Register”.

---

## UC-02-AT-02 — Guest can view price list (Main Success Path)
**Priority:** P0  
**Test Data:** DS-01  
**Preconditions:** Pricing exists in DB; guest is not logged in.  
**Steps:**
1. Navigate to the homepage.
2. Click the “Pricing / Registration Prices” link.
**Expected Results:**
- Pricing page loads successfully.
- Price list is displayed in a readable format.
- Prices are shown **by attendance type** (each type clearly labeled).
- No authentication prompt or redirect occurs.

---

## UC-02-AT-03 — Direct URL access to pricing works for guests
**Priority:** P1  
**Test Data:** DS-01  
**Preconditions:** Pricing page has a direct route (e.g., `/pricing` or `/registration/prices`).  
**Steps:**
1. Enter the pricing URL directly in the address bar.
**Expected Results:**
- Pricing page loads and displays the price list.
- No authentication required.

---

## UC-02-AT-04 — Displayed prices match database values
**Priority:** P1  
**Test Data:** DS-01  
**Preconditions:** DS-01 is stored in database.  
**Steps:**
1. Open the pricing page.
2. Compare displayed attendance types and prices to DS-01.
**Expected Results:**
- Each configured attendance type appears exactly once.
- Each displayed price matches the stored value.
- No missing or duplicate entries relative to DS-01.

---

## UC-02-AT-05 — Pricing unavailable message when not configured (Extension 4a)
**Priority:** P0  
**Test Data:** DS-02  
**Preconditions:** Pricing is not available (no records or not published).  
**Steps:**
1. Navigate to the pricing page as a guest.
**Expected Results:**
- Page loads successfully.
- CMS displays a clear message such as:
  - “Registration pricing is currently unavailable.”
- No raw errors, stack traces, or broken layout.

---

## UC-02-AT-06 — Database unavailable shows friendly error (Extension 4b)
**Priority:** P0  
**Preconditions:** Simulate DB outage/unavailability (stop DB or force connection failure).  
**Steps:**
1. Navigate to the pricing page as a guest.
**Expected Results:**
- CMS displays a user-friendly error message:
  - “Pricing information cannot be retrieved at this time.”
- No sensitive technical details are exposed (no stack traces, SQL, credentials).
- Guest is not redirected to login.

---

## UC-02-AT-07 — Web server unavailable behavior (Extension 1a)
**Priority:** P0  
**Preconditions:** Simulate CMS web server down/unreachable.  
**Steps:**
1. Attempt to load the CMS homepage or pricing page.
**Expected Results:**
- User receives an unavailability response (browser error or CMS error page).
- If CMS error page is shown, it indicates temporary service unavailability.

---

## UC-02-AT-08 — Pricing page does not expose restricted admin controls
**Priority:** P2  
**Test Data:** DS-01  
**Preconditions:** Guest is not logged in.  
**Steps:**
1. Navigate to pricing page.
2. Look for controls such as “Edit Prices”, “Add Price Type”, “Delete”.
**Expected Results:**
- No admin-only pricing management controls are visible to the guest.
- Guest can only view pricing information.

---

## UC-02-AT-09 — Cross-browser check (Chrome & Firefox)
**Priority:** P1  
**Test Data:** DS-01  
**Preconditions:** Guest is not logged in.  
**Steps:**
1. Run UC-02-AT-02 in Chrome.
2. Run UC-02-AT-02 in Firefox.
**Expected Results:**
- Pricing page loads and renders correctly in both browsers.
- Values and attendance type labels are consistent.
