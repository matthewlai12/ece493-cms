# Acceptance Test Suite — UC-23 Generate Conference Schedule

## Overview
**Use Case:** UC-23 Generate Conference Schedule  
**Primary Actor:** Administrator  
**Objective:** Verify that the CMS can automatically generate a valid, conflict-free conference schedule using accepted papers and defined constraints, store it correctly, display it for review, and handle all failure and cancellation scenarios safely.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- Only administrators can generate schedules.
- A scheduling algorithm is implemented and callable by the CMS.
- A “generated schedule” is stored in a draft/unpublished state.
- Schedule constraints include:
  - No overlapping sessions in the same room
  - Valid room and time slot usage
  - All accepted papers are scheduled exactly once

### Seed Accounts / Entities
**ADM-01 Administrator**
- Email/Username: `admin_user@example.com`
- Password: `ValidPass!234`

**ED-01 Editor**
- Email/Username: `editor_user@example.com`

### Paper Data
**Accepted Papers Set**
- Papers: P-01 … P-10
- Status: Accepted

### Scheduling Resources
- Rooms: Room A, Room B, Room C
- Time Slots: Day 1 (09:00–17:00), Day 2 (09:00–17:00)
- Constraints: No room/time overlaps, capacity sufficient

---

## UC-23-AT-01 — Administrator can access schedule generation option (Main Success Path)
**Priority:** P0  
**Preconditions:** Administrator authenticated; accepted papers exist.  
**Steps:**
1. Log in as administrator (ADM-01).
2. Navigate to the schedule management area.
**Expected Results:**
- CMS displays an option to generate the conference schedule.
- Generation controls are enabled.

---

## UC-23-AT-02 — Successful schedule generation with valid input (Main Success Path)
**Priority:** P0  
**Test Data:** ADM-01 + accepted papers + valid constraints  
**Preconditions:** All required scheduling data is present.  
**Steps:**
1. Select “Generate Conference Schedule”.
2. Confirm generation request.
**Expected Results:**
- CMS retrieves accepted papers and constraints.
- CMS executes the scheduling algorithm.
- A complete, conflict-free schedule is generated.
- CMS stores the generated schedule in the database.
- CMS displays the schedule to the administrator.

---

## UC-23-AT-03 — Generated schedule satisfies basic constraints
**Priority:** P0  
**Preconditions:** A schedule has been generated successfully.  
**Steps:**
1. Review the generated schedule.
2. Check session times and room assignments.
**Expected Results:**
- No two sessions overlap in the same room.
- Each accepted paper appears exactly once.
- All sessions are within defined time slots.

---

## UC-23-AT-04 — Missing scheduling data prevents generation (Extension 4a)
**Priority:** P0  
**Preconditions:** Remove or invalidate required data (e.g., no rooms defined).  
**Steps:**
1. Attempt to generate schedule.
**Expected Results:**
- CMS detects missing/incomplete scheduling data.
- CMS displays a clear error message indicating what is missing.
- Schedule generation is aborted.
- No schedule is stored.

---

## UC-23-AT-05 — Scheduling algorithm fails to find a valid schedule (Extension 5a)
**Priority:** P0  
**Preconditions:** Create conflicting constraints (e.g., too many papers for available slots).  
**Steps:**
1. Attempt to generate schedule.
**Expected Results:**
- CMS executes the scheduling algorithm.
- Algorithm fails to find a valid solution.
- CMS displays an error stating no valid schedule could be generated.
- No schedule is stored.

---

## UC-23-AT-06 — Database unavailable prevents storing generated schedule (Extension 7a)
**Priority:** P0  
**Preconditions:** Simulate CMS database outage.  
**Steps:**
1. Attempt to generate schedule.
**Expected Results:**
- CMS may generate schedule in memory.
- CMS fails to store the schedule.
- CMS displays an error indicating save failure.
- No partial schedule is persisted.

---

## UC-23-AT-07 — Administrator cancels schedule generation (Extension 3a)
**Priority:** P1  
**Preconditions:** Schedule generation process is started (e.g., long-running).  
**Steps:**
1. Start schedule generation.
2. Select “Cancel” before completion.
**Expected Results:**
- CMS stops the generation process.
- No schedule is stored.
- Administrator is returned to schedule management area.

---

## UC-23-AT-08 — Web server unavailable behavior (Extension 1a)
**Priority:** P0  
**Preconditions:** Simulate CMS web server outage.  
**Steps:**
1. Attempt to access or initiate schedule generation.
**Expected Results:**
- CMS displays system error page.
- No schedule is generated or stored.

---

## UC-23-AT-09 — Security: non-administrator cannot generate schedule
**Priority:** P0  
**Preconditions:** Non-admin account exists (e.g., editor or author).  
**Steps:**
1. Log in as non-admin user.
2. Attempt to access schedule generation functionality.
**Expected Results:**
- CMS denies access (403 or safe redirect).
- Generation controls are not visible or usable.

---

## UC-23-AT-10 — Generated schedule is available for later editing (integration with UC-22)
**Priority:** P1  
**Preconditions:** Schedule generated successfully.  
**Steps:**
1. Navigate to schedule management.
2. Select the generated schedule for editing.
**Expected Results:**
- CMS allows editor/administrator to open schedule in edit mode.
- Generated data is preserved accurately.
