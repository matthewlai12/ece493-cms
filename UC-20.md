# Use Case UC-20: Receive Completed Review Forms

**Goal in Context**:  
Enable an editor to receive and access completed review forms for submitted papers so that informed acceptance or rejection decisions can be made.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Editor

**Secondary Actors**:  
Referee  
CMS Web Interface  
CMS Database  
Notification Service

**Trigger**:  
A referee submits a completed review form for an assigned paper.

---

## Success End Condition
* The editor successfully receives and can view all completed review forms associated with a paper.

## Failed End Condition
* The editor cannot access one or more completed review forms, preventing proper evaluation of the paper.

---

## Preconditions
* The editor is registered and authenticated in the CMS.
* The paper has been assigned to one or more referees.
* At least one referee has submitted a completed review form.
* The CMS review storage and notification mechanisms are available.

---

## Main Success Scenario
1. A referee submits a completed review form for an assigned paper.
2. The CMS stores the review form in the database.
3. The CMS associates the review form with the corresponding paper.
4. The CMS notifies the editor that a review form has been submitted.
5. The editor logs into the CMS.
6. The editor navigates to the list of submitted papers.
7. The editor selects a paper with completed reviews.
8. The CMS displays the completed review form(s) for the selected paper.

---

## Extensions

* **1a**: Review form submission is incomplete or invalid  
    * **1a1**: The CMS rejects the submission and does not notify the editor.

* **2a**: CMS database is unavailable  
    * **2a1**: The CMS displays an error message indicating that the review form could not be stored.

* **4a**: Notification delivery fails  
    * **4a1**: The CMS logs the failure and allows the editor to view the review forms upon manual access.

* **8a**: Review form data is unavailable or corrupted  
    * **8a1**: The CMS displays an error message indicating that the review cannot be retrieved.

* **5a**: Editor is not authenticated  
    * **5a1**: The CMS prompts the editor to log in before accessing review forms.

* **1b**: CMS web server is unavailable  
    * **1b1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent during the paper evaluation phase  
* **Open Issues**:  
  * Whether editors should be notified after each review or only when all required reviews are complete  
  * Whether editors should be able to annotate or comment on review forms within the system
