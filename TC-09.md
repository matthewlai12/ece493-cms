# Acceptance Test Suite — UC-09 Upload Manuscript File

## Overview
**Use Case:** UC-09 Upload Manuscript File  
**Primary Actor:** Author  
**Objective:** Verify that an authenticated author can upload a compliant manuscript file during submission, and that invalid formats, oversized files, cancellations, and system failures are handled correctly.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- Allowed file formats: **PDF, Word (.doc/.docx), LaTeX (.tex or approved package)**.
- Maximum file size: **≤ 7 MB**.
- Upload occurs as part of an active paper submission (UC-08 context).

### Seed Account
**DS-01 (Valid author account)**
- Email/Username: `author_user@example.com`
- Password: `ValidPass!234`
- Role: Author

### File Test Sets
**F-01 Valid PDF (≤ 7MB)**: `manuscript_valid.pdf`  
**F-02 Valid Word (≤ 7MB)**: `manuscript_valid.docx`  
**F-03 Valid LaTeX (≤ 7MB)**: `manuscript_valid.tex`  
**F-04 Unsupported format**: `manuscript_invalid.png`  
**F-05 Oversized valid type (> 7MB)**: `manuscript_oversize.pdf`

## UC-09-AT-01 — Upload option is available during submission
**Priority:** P0  
**Preconditions:** Author is logged in (DS-01) and has started a paper submission.  
**Steps:**
1. Navigate to the paper submission form.
2. Locate the manuscript upload control.
**Expected Results:**
- Upload option is visible and enabled.
- Author can initiate file selection.

---

## UC-09-AT-02 — Successful upload of valid PDF file (Main Success Path)
**Priority:** P0  
**Test Data:** DS-01 + F-01  
**Preconditions:** Author is logged in and on submission form.  
**Steps:**
1. Select the upload option.
2. Choose `manuscript_valid.pdf`.
3. Confirm upload.
**Expected Results:**
- CMS validates file format and size.
- File is stored in file storage system.
- File is associated with the current paper submission.
- CMS displays confirmation that upload was successful.

---

## UC-09-AT-03 — Successful upload of other valid formats
**Priority:** P1  
**Test Data:** DS-01 + F-02 / F-03  
**Preconditions:** Author is logged in and on submission form.  
**Steps (repeat per format):**
1. Upload a valid Word or LaTeX file.
**Expected Results:**
- CMS accepts each supported format.
- File is stored and associated with the submission.

## UC-09-AT-06 — File storage unavailable blocks upload (Extension 7a)
**Priority:** P0  
**Test Data:** DS-01 + F-01  
**Preconditions:** Simulate file storage system outage/unavailability.  
**Steps:**
1. Attempt to upload a valid file.
**Expected Results:**
- CMS displays an error indicating upload failed.
- File is not stored.
- No file association is created.

---

## UC-09-AT-07 — Author cancels file upload (Extension 4a)
**Priority:** P2  
**Preconditions:** Author is logged in and upload dialog is open.  
**Steps:**
1. Open file chooser.
2. Cancel file selection (or cancel upload action).
**Expected Results:**
- CMS aborts upload process.
- Author is returned to submission form.
- No file is stored or associated.

---

## UC-09-AT-08 — Only one manuscript file is associated at a time
**Priority:** P1  
**Test Data:** DS-01 + F-01 + F-02  
**Preconditions:** Author is logged in and on submission form.  
**Steps:**
1. Upload F-01 successfully.
2. Upload F-02 as a replacement.
**Expected Results:**
- CMS associates the most recent file with the submission.
- Previous file is either replaced or marked obsolete (per design).
- No multiple active manuscript files are attached simultaneously.

---

## UC-09-AT-09 — Unauthenticated access to upload is blocked
**Priority:** P1  
**Preconditions:** User is logged out.  
**Steps:**
1. Attempt to access submission/upload page directly.
**Expected Results:**
- CMS redirects to login or displays access denied.
- Upload functionality is not accessible.
