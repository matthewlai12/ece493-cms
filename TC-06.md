# Acceptance Test Suite — UC-06 Log In to System

## Overview
**Use Case:** UC-06 Log In to System  
**Primary Actor:** Registered User  
**Objective:** Verify registered users can authenticate with valid credentials and reach their dashboard, and that all alternative paths (invalid credentials, missing account, locked account, DB/server failure) behave correctly.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- Base URL: `https://<cms-host>/`
- Login page is reachable via homepage “Log In” and/or direct route (e.g., `/login`).
- A “personalized dashboard” exists and is accessible only after successful authentication.

### Seed Accounts / Data Sets
**DS-01 (Valid active account)**
- Email/Username: `active_user@example.com`
- Password: `ValidPass!234`
- Status: Active (not locked/disabled)

**DS-02 (Nonexistent account)**
- Email/Username: `no_such_user@example.com`
- Password: `AnyPass!234`

**DS-03 (Wrong password)**
- Email/Username: `active_user@example.com`
- Password: `WrongPass!999`

**DS-04 (Locked/disabled account)**
- Email/Username: `locked_user@example.com`
- Password: `ValidPass!234`
- Status: Locked or Disabled

---

## UC-06-AT-01 — Login option is available from homepage
**Priority:** P0  
**Preconditions:** CMS deployed and reachable.  
**Steps:**
1. Open the CMS homepage.
2. Locate “Log In”.
3. Click “Log In”.
**Expected Results:**
- Homepage loads successfully (HTTP 200).
- “Log In” option is visible.
- Clicking “Log In” navigates to the login form.

---

## UC-06-AT-02 — Login form displays required fields
**Priority:** P0  
**Preconditions:** Login page accessible.  
**Steps:**
1. Navigate to the login page.
2. Inspect the form.
**Expected Results:**
- Login form is displayed.
- Fields include:
  - Email/Username
  - Password
- A submit control exists (e.g., “Log In”).

---

## UC-06-AT-03 — Successful login redirects to dashboard (Main Success Path)
**Priority:** P0  
**Test Data:** DS-01  
**Preconditions:** DS-01 account exists and is active.  
**Steps:**
1. Navigate to login form.
2. Enter DS-01 email/username and password.
3. Submit login.
**Expected Results:**
- CMS validates credentials against stored user record.
- User is authenticated successfully.
- CMS redirects user to their personalized dashboard.
- Dashboard indicates authenticated session (e.g., user name visible, logout button available).

---

## UC-06-AT-04 — Authenticated user can access a protected page
**Priority:** P1  
**Test Data:** DS-01  
**Preconditions:** Successful login (UC-06-AT-03).  
**Steps:**
1. After login, attempt to access a protected feature/page (e.g., submission dashboard, profile).
**Expected Results:**
- Protected feature/page loads successfully.
- No redirect to login occurs.

---

## UC-06-AT-05 — Incorrect password shows invalid credentials error (Extension 5a)
**Priority:** P0  
**Test Data:** DS-03  
**Preconditions:** DS-01 account exists and is active.  
**Steps:**
1. Navigate to login form.
2. Enter `active_user@example.com` with DS-03 incorrect password.
3. Submit login.
**Expected Results:**
- CMS denies authentication.
- CMS displays an error message indicating invalid credentials.
- User remains on login page and can retry.
- No dashboard access is granted.

---

## UC-06-AT-06 — Nonexistent account shows account-not-found message (Extension 6a)
**Priority:** P0  
**Test Data:** DS-02  
**Preconditions:** `no_such_user@example.com` does not exist.  
**Steps:**
1. Navigate to login form.
2. Enter DS-02 email/username and password.
3. Submit login.
**Expected Results:**
- CMS denies authentication.
- CMS displays an error indicating account was not found and suggests registration.
- User remains on login page (or is offered a link to register).

---

## UC-06-AT-07 — Locked/disabled account cannot log in (Extension 6b)
**Priority:** P0  
**Test Data:** DS-04  
**Preconditions:** DS-04 account exists and is locked/disabled.  
**Steps:**
1. Navigate to login form.
2. Enter DS-04 email/username and password.
3. Submit login.
**Expected Results:**
- CMS denies authentication.
- CMS displays message indicating account is locked/disabled and advises contacting support.
- No dashboard access is granted.

---

## UC-06-AT-08 — Database unavailable blocks authentication (Extension 6c)
**Priority:** P0  
**Preconditions:** Simulate DB outage/unavailability (stop DB or force connection failure).  
**Steps:**
1. Navigate to login form.
2. Enter valid credentials (DS-01).
3. Submit login.
**Expected Results:**
- CMS displays an error message indicating authentication cannot be completed at this time.
- No sensitive technical details exposed (no stack traces, SQL, credentials).
- User is not authenticated; no dashboard access.

---

## UC-06-AT-09 — Web server unavailable behavior (Extension 1a)
**Priority:** P0  
**Preconditions:** Simulate CMS web server down/unreachable.  
**Steps:**
1. Attempt to load homepage or login page.
**Expected Results:**
- User receives unavailability response (browser error or CMS error page).
- If CMS error page is shown, it indicates temporary service unavailability.

---

## UC-06-AT-10 — Session state persists across page refresh after login
**Priority:** P2  
**Test Data:** DS-01  
**Preconditions:** Successful login (UC-06-AT-03).  
**Steps:**
1. Refresh the dashboard page.
2. Navigate to another protected page.
**Expected Results:**
- User remains authenticated.
- No redirect to login occurs due to refresh/navigation.

---

## UC-06-AT-11 — Cross-browser login behavior is consistent
**Priority:** P1  
**Test Data:** DS-01 and DS-03  
**Preconditions:** None.  
**Steps:**
1. Run UC-06-AT-03 in Chrome.
2. Run UC-06-AT-05 in Firefox.
**Expected Results:**
- Successful login works in both browsers.
- Error handling for invalid credentials is consistent.
