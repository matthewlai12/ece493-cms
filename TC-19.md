# Acceptance Test Suite — UC-19 Ensure Three Reviewers per Paper

## Overview
**Use Case:** UC-19 Ensure Three Reviewers per Paper  
**Primary Actor:** Editor  
**Objective:** Verify the CMS enforces the constraint that each submitted paper must have **exactly three** referees assigned, informs the editor when fewer than three are assigned, and blocks assignments beyond three. Also verify workload limit interactions and DB/server failure handling.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- The system enforces **max reviewers per paper = 3**.
- The system can display the current reviewer count for a paper.
- Referee workload limit exists (max assigned papers per referee = 5) and may prevent assignment even when fewer than three reviewers exist.

## UC-19-AT-01 — Editor can view current assigned referees for a paper
**Priority:** P0  
**Preconditions:** Editor authenticated; paper exists (P-01/P-02/P-03).  
**Steps:**
1. Log in as editor (E-01).
2. Open submitted paper P-01 (repeat for P-02/P-03).
3. Open referee assignment/reviewer list section.
**Expected Results:**
- CMS displays the list of currently assigned referees (possibly empty).
- CMS displays an accurate reviewer count for the paper.

---

## UC-19-AT-02 — Paper with fewer than 3 reviewers prompts editor to add more (Extension 4a)
**Priority:** P0  
**Test Data:** E-01 + P-02 (2 reviewers)  
**Preconditions:** P-02 has exactly 2 assigned referees.  
**Steps:**
1. Log in as editor.
2. Open paper P-02 reviewer section.
**Expected Results:**
- CMS informs editor that additional referees must be assigned before reviews can begin / requirement not satisfied.
- CMS indicates current count (2/3) and that 1 more is needed (wording may vary).

---

## UC-19-AT-03 — Assigning a third reviewer satisfies the requirement (Main Success Path)
**Priority:** P0  
**Test Data:** E-01 + P-02 + R-03  
**Preconditions:** P-02 has 2 reviewers; R-03 eligible (below workload max).  
**Steps:**
1. Log in as editor.
2. Open paper P-02 assignment interface.
3. Assign R-03 as an additional reviewer.
4. Refresh/reopen reviewer list.
**Expected Results:**
- CMS counts reviewers and determines total is exactly 3.
- CMS confirms the reviewer requirement is satisfied.
- Paper now shows 3/3 reviewers assigned.
- Review process can proceed (e.g., invitations issued / status updated per design).

---

## UC-19-AT-04 — Attempting to add a fourth reviewer is blocked (Extension 4b)
**Priority:** P0  
**Test Data:** E-01 + P-03 + R-04  
**Preconditions:** P-03 already has 3 assigned referees.  
**Steps:**
1. Log in as editor.
2. Open paper P-03 assignment interface.
3. Attempt to assign R-04.
**Expected Results:**
- CMS blocks the additional assignment.
- CMS informs editor that the maximum number of referees has been reached (3).
- No new assignment record is created; count remains 3.

---

## UC-19-AT-05 — Workload limit prevents selecting a reviewer even when fewer than 3 exist (Extension 3a)
**Priority:** P0  
**Test Data:** E-01 + P-02 + R-05  
**Preconditions:** P-02 has fewer than 3 reviewers; R-05 is at workload limit.  
**Steps:**
1. Log in as editor.
2. Open paper P-02 assignment interface.
3. Attempt to assign R-05.
**Expected Results:**
- CMS prevents assignment of R-05.
- CMS informs editor of the workload constraint.
- Reviewer count for the paper does not change.

---

## UC-19-AT-06 — Trigger via referee acceptance does not allow exceeding 3 reviewers
**Priority:** P1  
**Preconditions:** Paper currently has 3 accepted/assigned reviewers; an additional pending invitation exists for a 4th referee (edge case).  
**Steps:**
1. Ensure paper has 3 reviewers already assigned/accepted.
2. Log in as the 4th referee and attempt to accept the invitation.
**Expected Results:**
- System blocks acceptance OR records acceptance but flags editor (depending on system rule design).
- **For UC-19 strictness**: the system must prevent the paper from ending with >3 assigned reviewers.
- Editor is informed if any constraint violation is detected.

---

## UC-19-AT-07 — Database unavailable prevents verifying reviewer count (Extension 2a)
**Priority:** P0  
**Test Data:** E-01 + P-02  
**Preconditions:** Simulate DB outage/unavailability.  
**Steps:**
1. Log in as editor.
2. Open paper reviewer list or attempt to assign a referee while DB is unavailable.
**Expected Results:**
- CMS displays an error indicating referee assignments cannot be verified.
- CMS prevents assignments until system becomes available.
- No partial/incorrect count is displayed as confirmed.

---

## UC-19-AT-08 — Web server unavailable behavior (Extension 1a)
**Priority:** P0  
**Preconditions:** Simulate CMS web server down/unreachable.  
**Steps:**
1. Attempt to access reviewer list or assign reviewers.
**Expected Results:**
- User receives unavailability response (browser error or CMS error page).
- No assignments occur.

---

## UC-19-AT-09 — Security: non-editor cannot assign reviewers or view restricted assignment controls
**Priority:** P0  
**Preconditions:** Non-editor account exists (e.g., referee/author).  
**Steps:**
1. Log in as a non-editor user.
2. Attempt to access paper reviewer assignment interface (UI or direct URL).
**Expected Results:**
- CMS denies access to assignment controls (403/Not authorized or safe redirect).
- Reviewer assignment cannot be performed.

---

## UC-19-AT-10 — Cross-browser reviewer-count enforcement is consistent
**Priority:** P2  
**Test Data:** E-01 + P-03 + R-04  
**Preconditions:** P-03 has 3 reviewers.  
**Steps:**
1. Attempt to add a 4th reviewer in Chrome.
2. Attempt to add a 4th reviewer in Firefox (reset state or use another paper already at 3).
**Expected Results:**
- Both browsers experience identical constraint enforcement and messaging.
