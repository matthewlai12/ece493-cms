# Use Case UC-03: View Conference Schedule

**Goal in Context**:  
Allow a guest user to view the published conference schedule without registering or logging into the system.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Public User (Guest)

**Secondary Actors**:  
CMS Web Interface  
CMS Database

**Trigger**:  
The guest navigates to the conference schedule section of the CMS website.

---

## Success End Condition
* The published conference schedule is successfully displayed to the guest.

## Failed End Condition
* The conference schedule is not displayed and the guest cannot view session information.

---

## Preconditions
* The CMS web application is deployed and accessible via a supported web browser.
* A final conference schedule has been generated and published in the CMS.

---

## Main Success Scenario
1. The guest accesses the CMS homepage.
2. The CMS displays navigation options, including a link to the conference schedule.
3. The guest selects the conference schedule option.
4. The CMS retrieves the published conference schedule from the database.
5. The CMS displays the conference schedule in HTML format to the guest.

---

## Extensions

* **4a**: Conference schedule has not been published  
    * **4a1**: The CMS displays a message indicating that the conference schedule is not yet available.

* **4b**: CMS database is unavailable  
    * **4b1**: The CMS displays an error message stating that the conference schedule cannot be retrieved at this time.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: Medium  
* **Frequency**: Frequent (guests and attendees often consult the schedule)  
* **Open Issues**:  
  * Whether the schedule should be searchable or filterable  
  * Whether schedule updates should be highlighted to users
