# Use Case UC-17: Enforce Reviewer Workload Limit

**Goal in Context**:  
Ensure that a referee is not assigned more than five papers to review in order to maintain a manageable and fair workload.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Referee

**Secondary Actors**:  
Editor  
CMS Web Interface  
CMS Database

**Trigger**:  
The editor attempts to assign a paper to a referee, or a referee attempts to accept a new review invitation.

---

## Success End Condition
* The referee is assigned no more than five papers, and any attempt to exceed this limit is prevented by the system.

## Failed End Condition
* The referee is assigned more than five papers, resulting in an unreasonable workload (constraint violation).

---

## Preconditions
* The referee is registered in the CMS.
* The referee has accepted zero or more review assignments.
* The CMS maintains an accurate count of papers assigned to each referee.

---

## Main Success Scenario
1. The editor selects a referee to assign to a submitted paper, or the referee selects an invitation to accept.
2. The CMS retrieves the current number of papers assigned to the referee.
3. The CMS determines that assigning the paper will not exceed the maximum limit of five papers.
4. The CMS allows the assignment or acceptance to proceed.
5. The CMS updates the referee’s list of assigned papers.
6. The CMS confirms that the assignment has been successfully recorded.

---

## Extensions

* **3a**: Assigning the paper would exceed the maximum of five papers  
    * **3a1**: The CMS blocks the assignment or acceptance.
    * **3a2**: The CMS displays a message indicating that the referee has reached the maximum review limit.
    * **3a3**: The CMS informs the editor that the referee cannot be assigned additional papers.

* **2a**: CMS database is unavailable  
    * **2a1**: The CMS displays an error message indicating that the referee’s workload cannot be verified.
    * **2a2**: The CMS prevents the assignment until the system becomes available.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent during referee assignment and invitation acceptance phases  
* **Open Issues**:  
  * Whether the maximum review limit should be configurable  
  * Whether editors should be allowed to override the limit in exceptional cases
