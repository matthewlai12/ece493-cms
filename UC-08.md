# Use Case UC-08: Submit Paper Manuscript

**Goal in Context**:  
Allow an author to submit a complete paper manuscript and required metadata so that the paper enters the conference review process.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Author

**Secondary Actors**:  
CMS Web Interface  
CMS Database  
File Storage System

**Trigger**:  
The author selects the “Submit Paper” option from their dashboard.

---

## Success End Condition
* The paper manuscript and all required submission information are successfully stored and marked as submitted for review.

## Failed End Condition
* The paper manuscript is not submitted, and no incomplete or invalid submission is recorded as final.

---

## Preconditions
* The author is registered in the CMS.
* The author is authenticated and logged into the CMS.
* The paper submission feature is currently available.

---

## Main Success Scenario
1. The author logs into the CMS.
2. The author selects the option to submit a new paper.
3. The CMS displays the paper submission form.
4. The author enters required paper metadata, including author names, affiliations, contact information, abstract, keywords, and main source.
5. The author uploads the manuscript file.
6. The author submits the completed paper submission.
7. The CMS validates all required fields and the uploaded manuscript file.
8. The CMS stores the paper metadata and manuscript file.
9. The CMS confirms successful submission and redirects the author to their dashboard.

---

## Extensions

* **5a**: Uploaded file format is not supported  
    * **5a1**: The CMS displays an error message indicating allowed formats (PDF, Word, LaTeX) and requests a valid file.

* **5b**: Uploaded file exceeds the maximum allowed size  
    * **5b1**: The CMS displays an error message indicating the file size limit has been exceeded and requests a smaller file.

* **7a**: Required submission fields are missing or invalid  
    * **7a1**: The CMS highlights invalid or incomplete fields and requests correction before submission can proceed.

* **8a**: Database or file storage system is unavailable  
    * **8a1**: The CMS displays an error message indicating that the submission could not be saved and does not finalize the submission.

* **6a**: Author cancels the submission process  
    * **6a1**: The CMS abandons the submission and returns the author to their dashboard without saving a final submission.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent during the paper submission period  
* **Open Issues**:  
  * Whether submission deadlines should be enforced automatically  
  * Whether authors can edit submissions after final submission
