# Acceptance Test Suite — UC-29 View Conference Schedule

## Overview
**Use Case:** UC-29 View Conference Schedule  
**Primary Actor:** Attendee  
**Objective:** Verify that an attendee can successfully view the published conference schedule, and that the system correctly handles authentication rules, unavailable schedules, and system failures.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- The conference schedule has already been **finalized and published** (UC-25 completed).
- Schedule may be publicly accessible or restricted to authenticated attendees (configurable).
- Schedule is stored in the CMS database and rendered in a readable format (HTML/table/list).

## UC-29-AT-01 — Attendee can access conference schedule page (Main Success Path)
**Priority:** P0  
**Preconditions:** Schedule published; CMS available.  
**Steps:**
1. Log in as attendee (if authentication is required).

3. Select the “Conference Schedule” option.
**Expected Results:**
- CMS navigates to the conference schedule page.
- No authentication or authorization errors occur.

## UC-29-AT-02 — Published schedule is displayed correctly (Main Success Path)
**Priority:** P0  
**Preconditions:** Schedule page accessible.  
**Steps:**

**Expected Results:**

- Schedule is displayed in a readable format.
- Sessions are shown with correct dates, times, and locations.

## UC-29-AT-03 — Attendee can review schedule content (Main Success Path)

**Preconditions:** Schedule displayed.

2. Review sessions across different days/times.

## UC-29-AT-04 — Authentication required but attendee not logged in (Extension 2a)
**Priority:** P0  
**Preconditions:** Schedule access restricted to authenticated users; attendee logged out.  
**Steps:**
1. Attempt to access the conference schedule page directly.
**Expected Results:**
- CMS prompts the attendee to log in.
- Schedule is not displayed until authentication succeeds.

## UC-29-AT-05 — Schedule not yet published (Extension 3a)
**Priority:** P0  
**Preconditions:** Schedule status set to “Unpublished.”  
**Steps:**
1. Navigate to the conference schedule page.
**Expected Results:**
- CMS displays a message indicating the schedule is not yet available.
- No schedule data is displayed.

## UC-29-AT-06 — CMS database unavailable (Extension 3b)
**Priority:** P0

**Expected Results:**

## UC-29-AT-07 — CMS web server unavailable (Extension 1a)
