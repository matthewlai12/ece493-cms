# Use Case UC-14: Accept or Reject Review Invitation

**Goal in Context**:  
Allow a referee to respond to a review invitation by either accepting or rejecting it so that they can manage their review workload effectively.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Referee

**Secondary Actors**:  
Editor  
CMS Web Interface  
CMS Database

**Trigger**:  
The referee selects a pending review invitation.

---

## Success End Condition
* The referee’s response (accept or reject) is successfully recorded, and the system updates the review assignment status accordingly.

## Failed End Condition
* The referee’s response is not recorded, and the review invitation remains unresolved.

---

## Preconditions
* The referee is registered in the CMS.
* The referee has received at least one review invitation.
* The CMS invitation management functionality is available.

---

## Main Success Scenario
1. The referee logs into the CMS.
2. The referee navigates to their list of pending review invitations.
3. The referee selects a review invitation.
4. The CMS displays the paper details and response options.
5. The referee chooses to accept or reject the invitation.
6. The CMS records the referee’s response.
7. If accepted, the CMS assigns the paper to the referee’s review list.
8. If rejected, the CMS marks the invitation as declined and notifies the editor.

---

## Extensions

* **5a**: Referee accepts the invitation but has reached the maximum review limit  
    * **5a1**: The CMS prevents acceptance and displays a message indicating the workload limit has been reached.

* **5b**: Referee attempts to accept an invitation that would exceed the maximum reviewers per paper  
    * **5b1**: The CMS records the acceptance but informs the editor that the reviewer limit has been exceeded.

* **6a**: CMS database is unavailable  
    * **6a1**: The CMS displays an error message indicating the response could not be recorded.

* **5c**: Referee cancels the response action  
    * **5c1**: The CMS returns the referee to the invitation list without recording a response.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent during the review invitation phase  
* **Open Issues**:  
  * Whether referees should be allowed to change their response after submission  
  * Whether automatic reassignment should occur when invitations are rejected
