# Use Case UC-16: Submit Paper Review

**Goal in Context**:  
Allow a referee to submit a completed review form for an assigned paper so that the evaluation is recorded in the system and delivered to the editor.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Referee

**Secondary Actors**:  
Editor  
CMS Web Interface  
CMS Database

**Trigger**:  
The referee selects the option to submit a review for an assigned paper.

---

## Success End Condition
* The completed review form is successfully stored in the CMS and made available to the editor.

## Failed End Condition
* The review form is not submitted, and the editor does not receive the referee’s evaluation.

---

## Preconditions
* The referee is registered and authenticated in the CMS.
* The referee has accepted a review invitation for the paper.
* The referee has access to the assigned paper.
* The review form template is available in the CMS.

---

## Main Success Scenario
1. The referee logs into the CMS.
2. The referee navigates to their list of assigned papers.
3. The referee selects a paper to review.
4. The CMS displays the review form for the selected paper.
5. The referee completes the review form.
6. The referee submits the completed review form.
7. The CMS validates the review form for completeness.
8. The CMS stores the review form in the database.
9. The CMS makes the submitted review available to the editor.
10. The CMS confirms successful submission to the referee.

---

## Extensions

* **7a**: Review form contains missing or invalid information  
    * **7a1**: The CMS displays validation errors and requests the referee to correct the form before submission.

* **6a**: Referee cancels review submission  
    * **6a1**: The CMS abandons the submission and returns the referee to the review form without saving.

* **8a**: CMS database is unavailable  
    * **8a1**: The CMS displays an error message indicating that the review could not be saved.

* **3a**: Referee attempts to submit a review for an unassigned paper  
    * **3a1**: The CMS blocks the action and displays an authorization error.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent during the paper review period  
* **Open Issues**:  
  * Whether referees should be allowed to edit reviews after submission  
  * Whether automatic reminders should be sent before review deadlines
