# Use Case UC-24: Modify Generated Conference Schedule

**Goal in Context**:  
Allow an administrator to modify an automatically generated conference schedule to accommodate real-world constraints such as room availability, presenter conflicts, or organizational requirements.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Administrator

**Secondary Actors**:  
Editor  
CMS Web Interface  
CMS Database

**Trigger**:  
The administrator selects an existing generated conference schedule for modification.

---

## Success End Condition
* The conference schedule is successfully modified, validated, and stored, reflecting real-world constraints.

## Failed End Condition
* The conference schedule is not modified, and real-world constraints remain unresolved.

---

## Preconditions
* The administrator is registered and authenticated in the CMS.
* A generated conference schedule exists.
* The CMS schedule editing functionality is available.

---

## Main Success Scenario
1. The administrator logs into the CMS.
2. The administrator navigates to the schedule management section.
3. The administrator selects a generated conference schedule.
4. The CMS displays the schedule in an editable format.
5. The administrator modifies schedule elements (e.g., session times, rooms, paper assignments).
6. The CMS validates the modified schedule against scheduling constraints.
7. The administrator submits the modified schedule.
8. The CMS saves the updated schedule to the database.
9. The CMS confirms that the schedule has been successfully modified.

---

## Extensions

* **6a**: Modified schedule violates constraints  
    * **6a1**: The CMS displays validation errors identifying conflicts or violations.
    * **6a2**: The CMS prevents saving and returns the administrator to the editing interface.

* **7a**: Administrator cancels schedule modification  
    * **7a1**: The CMS discards unsaved changes and restores the previous schedule.

* **8a**: CMS database is unavailable  
    * **8a1**: The CMS displays an error message indicating that schedule changes could not be saved.

* **3a**: Selected schedule does not exist  
    * **3a1**: The CMS displays an error message and returns the administrator to the schedule list.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: Medium–High  
* **Frequency**: Occasional, during schedule finalization  
* **Open Issues**:  
  * Whether administrators should be able to override all validation constraints  
  * Whether schedule modification history should be maintained for audit purposes
