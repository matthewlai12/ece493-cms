# Use Case UC-01: View Public Announcements

**Goal in Context**:  
Allow an unregistered user to view conference-related announcements without requiring authentication.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Public User (Guest)

**Secondary Actors**:  
CMS Web Interface  
CMS Database

**Trigger**:  
The guest navigates to the CMS website or selects the announcements section.

---

## Success End Condition
* Public announcements are successfully retrieved and displayed to the guest.

## Failed End Condition
* Public announcements are not displayed and the guest cannot view conference information.

---

## Preconditions
* The CMS web application is deployed and accessible via a supported web browser.
* Public announcements have been created and stored in the CMS database.

---

## Main Success Scenario
1. The guest accesses the CMS homepage.
2. The CMS displays available navigation options, including a link to public announcements.
3. The guest selects the public announcements option.
4. The CMS requests the list of public announcements from the database.
5. The CMS retrieves the announcements successfully.
6. The CMS displays the announcements to the guest in a readable format.

---

## Extensions

* **3a**: No public announcements exist  
    * **3a1**: The CMS displays a message indicating that no announcements are currently available.

* **4a**: Database is unavailable  
    * **4a1**: The CMS displays an error message stating that announcements cannot be retrieved at this time.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Very frequent (every visitor may trigger this use case)  
* **Open Issues**:  
  * Whether announcements should be categorized or searchable
