# Acceptance Test Suite — UC-03 View Conference Schedule

## Overview
**Use Case:** UC-03 View Conference Schedule  
**Primary Actor:** Public User (Guest)  
**Objective:** Verify guests can view the *published* conference schedule without authentication, and that all alternative paths behave correctly.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- Base URL: `https://<cms-host>/`
- The schedule is stored in the CMS database and rendered as an HTML page/section.
- A navigation link exists for “Schedule” or “Conference Schedule”.

### Seed Data Sets
**DS-01 (Published schedule exists)**
- At least 1 day of sessions, e.g.:
  - Day 1: 09:00–10:00 Session A (Room 101)
  - Day 1: 10:15–11:15 Session B (Room 102)
- The schedule is marked as **published**.

**DS-02 (Schedule not published)**
- Schedule exists in draft OR no schedule exists OR published flag is false.

---

## UC-03-AT-01 — Guest can access homepage and see schedule navigation
**Priority:** P0  
**Preconditions:** CMS deployed and reachable.  
**Steps:**
1. Open the CMS base URL in a supported browser.
**Expected Results:**
- Homepage loads successfully (HTTP 200).
- No login is required to view the homepage.
- Navigation includes a link to “Schedule” / “Conference Schedule”.

---

## UC-03-AT-02 — Guest can view published schedule (Main Success Path)
**Priority:** P0  
**Test Data:** DS-01  
**Preconditions:** Schedule is published; guest is not logged in.  
**Steps:**
1. Navigate to the homepage.
2. Click “Schedule / Conference Schedule”.
**Expected Results:**
- Schedule page loads successfully.
- Schedule is displayed to the guest in **HTML format** (rendered page, not raw markup).
- Content includes readable session information (at minimum: session title and time; optionally room/location).
- No authentication prompt or redirect occurs.

---

## UC-03-AT-03 — Direct URL access to schedule works for guests
**Priority:** P1  
**Test Data:** DS-01  
**Preconditions:** Schedule page has a direct route (e.g., `/schedule`).  
**Steps:**
1. Enter the schedule URL directly in the address bar.
**Expected Results:**
- Schedule page loads and shows published schedule.
- No authentication required.

---

## UC-03-AT-04 — Displayed schedule matches stored schedule data
**Priority:** P1  
**Test Data:** DS-01  
**Preconditions:** DS-01 schedule exists in the database and is published.  
**Steps:**
1. Open schedule page.
2. Verify key schedule entries (times/titles/rooms) match DS-01.
**Expected Results:**
- Session entries match the stored schedule (no missing/extra sessions relative to DS-01).
- Ordering is sensible (e.g., chronological by time within a day).

---

## UC-03-AT-05 — Schedule not published shows “not yet available” message (Extension 4a)
**Priority:** P0  
**Test Data:** DS-02  
**Preconditions:** Schedule is not published.  
**Steps:**
1. Navigate to schedule page as a guest.
**Expected Results:**
- Page loads successfully.
- CMS displays a clear message such as:
  - “The conference schedule is not yet available.”
- No broken layout or raw system error output.

---

## UC-03-AT-06 — Database unavailable shows friendly error (Extension 4b)
**Priority:** P0  
**Preconditions:** Simulate DB outage/unavailability (stop DB or force connection failure).  
**Steps:**
1. Navigate to the schedule page as a guest.
**Expected Results:**
- CMS displays a user-friendly error message:
  - “The conference schedule cannot be retrieved at this time.”
- No sensitive technical details exposed (no stack traces, SQL, credentials).
- Guest is not redirected to login.

---

## UC-03-AT-07 — Web server unavailable behavior (Extension 1a)
**Priority:** P0  
**Preconditions:** Simulate CMS web server down/unreachable.  
**Steps:**
1. Attempt to load the CMS homepage or schedule page.
**Expected Results:**
- User receives an unavailability response (browser error or CMS error page).
- If CMS error page is shown, it indicates temporary service unavailability.

---

## UC-03-AT-08 — Schedule page does not expose restricted editing controls to guests
**Priority:** P2  
**Test Data:** DS-01  
**Preconditions:** Guest is not logged in; schedule published.  
**Steps:**
1. Navigate to schedule page.
2. Look for controls such as “Edit schedule”, “Publish”, “Generate schedule”, “Admin”.
**Expected Results:**
- No schedule management controls are visible to guests.
- Guest can only view the schedule.

---

## UC-03-AT-09 — Cross-browser check (Chrome & Firefox)
**Priority:** P1  
**Test Data:** DS-01  
**Preconditions:** Guest is not logged in; schedule published.  
**Steps:**
1. Run UC-03-AT-02 in Chrome.
2. Run UC-03-AT-02 in Firefox.
**Expected Results:**
- Schedule loads and renders correctly in both browsers.
- Session content appears consistently.
