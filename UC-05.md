# Use Case UC-05: Validate Registration Information

**Goal in Context**:  
Ensure that all user-provided registration information is valid, complete, and secure before creating a user account.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Registered User (new user during registration)

**Secondary Actors**:  
CMS Web Interface  
CMS Database  
Validation Service

**Trigger**:  
The user submits the registration form.

---

## Success End Condition
* The user’s registration information is validated successfully, allowing the account creation process to continue.

## Failed End Condition
* The registration information is found to be invalid, and the user account is not created.

---

## Preconditions
* The CMS registration form is accessible.
* The user has entered registration information into the form.
* The CMS validation mechanisms are operational.

---

## Main Success Scenario
1. The user completes the registration form with required information.
2. The user submits the registration form.
3. The CMS validates the email format.
4. The CMS verifies that the email address is unique.
5. The CMS validates the password against security requirements.
6. The CMS checks that all required fields are completed.
7. The CMS confirms that the registration information is valid.
8. The CMS allows the registration process to proceed to account creation.

---

## Extensions

* **3a**: Email format is invalid  
    * **3a1**: The CMS displays an error message indicating the email format is invalid and requests correction.

* **4a**: Email address already exists  
    * **4a1**: The CMS displays an error message stating the email is already registered and prevents account creation.

* **5a**: Password does not meet security requirements  
    * **5a1**: The CMS displays an error message indicating password policy violations.

* **6a**: Required registration fields are missing  
    * **6a1**: The CMS highlights missing fields and requests completion before proceeding.

* **7a**: CMS database is unavailable  
    * **7a1**: The CMS displays an error message indicating validation cannot be completed at this time.

* **2a**: User cancels registration submission  
    * **2a1**: The CMS aborts the validation process and returns the user to the registration form.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent (occurs for every new user registration)  
* **Open Issues**:  
  * Whether email verification links should be required  
  * Whether password strength feedback should be shown in real time
