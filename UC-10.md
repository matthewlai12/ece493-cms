# Use Case UC-10: Save Submission Progress

**Goal in Context**:  
Allow an author to save a partially completed paper submission so that the submission can be resumed at a later time without loss of data.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Author

**Secondary Actors**:  
CMS Web Interface  
CMS Database

**Trigger**:  
The author selects the “Save” option during the paper submission process.

---

## Success End Condition
* The current state of the paper submission is successfully stored and can be retrieved later by the author.

## Failed End Condition
* The submission progress is not saved, and newly entered information is not retained.

---

## Preconditions
* The author is registered in the CMS.
* The author is authenticated and logged into the CMS.
* The author has initiated a paper submission.
* The CMS submission-saving functionality is available.

---

## Main Success Scenario
1. The author is completing the paper submission form.
2. The author selects the “Save” option.
3. The CMS validates the currently entered submission data.
4. The CMS stores the partial submission data in the database.
5. The CMS confirms that the submission progress has been saved.
6. The CMS allows the author to continue editing or exit the submission process.

---

## Extensions

* **3a**: Submission data contains invalid or incomplete values  
    * **3a1**: The CMS displays validation error messages and does not save the submission progress.

* **4a**: CMS database is unavailable  
    * **4a1**: The CMS displays an error message indicating that submission progress could not be saved.

* **2a**: Author cancels the save operation  
    * **2a1**: The CMS abandons the save operation and returns the author to the submission form without saving changes.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: Medium  
* **Frequency**: Frequent (authors may save progress multiple times during submission)  
* **Open Issues**:  
  * Whether automatic periodic saving (autosave) should be supported  
  * Whether partially saved submissions should expire after a deadline
