# Acceptance Test Suite — UC-08 Submit Paper Manuscript

## Overview
**Use Case:** UC-08 Submit Paper Manuscript  
**Primary Actor:** Author  
**Objective:** Verify an authenticated author can submit a complete paper (metadata + manuscript file) and that validation, cancellation, and failure modes behave correctly.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- Author must be authenticated to access submission features.
- Allowed file formats: **PDF, Word, LaTeX**.
- Maximum file size: **≤ 7 MB**.
- Required metadata fields include (per UC): author names, affiliations, contact information, abstract, keywords, main source.

### Seed Accounts / Data Sets
**DS-01 (Valid author account)**
- Email/Username: `author_user@example.com`
- Password: `ValidPass!234`
- Role: Author
- Status: Active

### Manuscript Files
**F-01 Valid PDF (≤ 7MB)**: `paper_valid.pdf`  
**F-02 Valid Word (≤ 7MB)**: `paper_valid.docx`  
**F-03 Valid LaTeX (≤ 7MB)**: `paper_valid.tex` (or zipped LaTeX package if your system requires)  
**F-04 Unsupported format**: `paper_invalid.txt` (or `.png`)  
**F-05 Oversized valid type (> 7MB)**: `paper_oversize.pdf`

### Metadata Sets
**M-01 Complete valid metadata**
- Author names: present
- Affiliations: present
- Contact info: valid email/phone if required
- Abstract: present
- Keywords: present
- Main source: present

**M-02 Missing/invalid metadata**
- Missing abstract OR missing keywords OR missing contact info (one variant per test)

## UC-08-AT-01 — Submit Paper option is visible to authenticated author
**Priority:** P0  
**Preconditions:** Author is logged in (DS-01). Submission feature is available.  
**Steps:**
1. Navigate to author dashboard.
2. Locate “Submit Paper”.
3. Click “Submit Paper”.
**Expected Results:**
- Submission form is displayed.
- Form includes required metadata fields and a manuscript upload control.

---

## UC-08-AT-02 — Successful paper submission with valid metadata + valid PDF (Main Success Path)
**Priority:** P0  
**Test Data:** DS-01 + M-01 + F-01  
**Preconditions:** Author is logged in; submission feature available.  
**Steps:**
1. Open the submission form.
2. Enter M-01 metadata.
3. Upload F-01 (valid PDF ≤ 7MB).
4. Submit the paper.
**Expected Results:**
- CMS validates all required metadata fields and file constraints.
- CMS stores paper metadata in the database.
- CMS stores the manuscript file in file storage and links it to the submission record.
- CMS confirms submission success.
- CMS redirects author to dashboard.

---

## UC-08-AT-03 — Submission persists and appears on author dashboard
**Priority:** P1  
**Test Data:** DS-01 + (result of AT-02)  
**Preconditions:** UC-08-AT-02 completed successfully.  
**Steps:**
1. Return to author dashboard (post-redirect).
2. Locate “My Submissions” (or equivalent).
3. Open the newly created submission entry.
**Expected Results:**
- Newly submitted paper appears in the author’s submission list.
- Submission status is “Submitted” (or equivalent review-queue state).
- Metadata is displayed correctly and file is associated.

---

## UC-08-AT-04 — Unsupported file format is rejected (Extension 5a)
**Priority:** P0  
**Test Data:** DS-01 + M-01 + F-04  
**Preconditions:** Author is logged in; submission form accessible.  
**Steps:**
1. Enter M-01 metadata.
2. Upload F-04 (unsupported format).
3. Submit.
**Expected Results:**
- CMS displays an error indicating allowed formats (PDF, Word, LaTeX).
- Submission is not finalized.
- User remains on the form and can replace the file.

## UC-08-AT-05 — Oversized file is rejected (Extension 5b)
**Priority:** P0  
**Test Data:** DS-01 + M-01 + F-05  
**Preconditions:** Author is logged in; submission form accessible.  
**Steps:**
1. Enter M-01 metadata.
2. Upload F-05 (> 7MB).
3. Submit.
**Expected Results:**
- CMS displays an error indicating the file size limit was exceeded (max 7MB).
- Submission is not finalized.
- User remains on the form and can upload a smaller file.

---

## UC-08-AT-06 — Missing required metadata blocks submission (Extension 7a)
**Priority:** P0  
**Test Data:** DS-01 + M-02 + F-01  
**Preconditions:** Author is logged in; submission form accessible.  
**Steps (repeat per missing/invalid field variant):**
1. Enter metadata with one required field missing/invalid (M-02 variant).
2. Upload F-01.
3. Submit.
**Expected Results:**
- CMS highlights missing/invalid fields and shows a corrective message.
- Submission is not finalized.
- Previously entered valid fields remain populated (no unnecessary data loss).

---

## UC-08-AT-07 — Multiple validation errors handled gracefully
**Priority:** P1  
**Test Data:** DS-01 + (M-02 variant) + F-04  
**Preconditions:** Author is logged in; submission form accessible.  
**Steps:**
1. Leave a required metadata field missing (M-02).
2. Upload unsupported file (F-04).
3. Submit.
**Expected Results:**
- CMS reports validation failures (either multiple messages or first blocking error, per implementation).
- Submission is not finalized.
- Author can correct and resubmit.

---

## UC-08-AT-08 — Storage unavailable prevents final submission (Extension 8a)
**Priority:** P0  
**Test Data:** DS-01 + M-01 + F-01  
**Preconditions:** Simulate file storage outage/unavailability at submission time.  
**Steps:**
1. Enter M-01 metadata.
2. Upload F-01.
3. Submit while storage is unavailable.
**Expected Results:**
- CMS displays a user-friendly error indicating the submission could not be saved.
- No final “Submitted” record is created.
- No orphaned/partial file is left accessible to the author (if partial upload occurred, it is not treated as submitted).

## UC-08-AT-10 — Author cancels submission (Extension 6a)
**Priority:** P2  
**Preconditions:** Author is logged in; cancel action exists.  
**Steps:**
1. Open submission form.
2. Enter some metadata and optionally upload a file.
3. Click “Cancel” (or provided cancel/back action).
**Expected Results:**
- CMS abandons the submission flow and returns author to dashboard.
- No final submission is created/marked “Submitted”.
- (If drafts are supported separately, any saved draft behavior must match UC-10, not UC-08.)

---

## UC-08-AT-11 — Unauthenticated access is blocked
**Priority:** P1  
**Preconditions:** User is logged out.  
**Steps:**
1. Attempt to navigate directly to the submission page route.
**Expected Results:**
- CMS redirects to login or shows access denied.
- Submission form is not accessible without authentication.

---

## UC-08-AT-12 — Web server unavailable behavior (Extension 1a)

**Expected Results:**
