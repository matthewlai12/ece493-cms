# Use Case UC-09: Upload Manuscript File

**Goal in Context**:  
Allow an author to upload a manuscript file that complies with the conference’s format and size requirements so it can be included in a paper submission.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Author

**Secondary Actors**:  
CMS Web Interface  
CMS Database  
File Storage System

**Trigger**:  
The author selects the option to upload a manuscript file during the paper submission process.

---

## Success End Condition
* The manuscript file is successfully validated, stored, and associated with the author’s paper submission.

## Failed End Condition
* The manuscript file is not stored, and no invalid or non-compliant file is associated with the submission.

---

## Preconditions
* The author is registered and authenticated in the CMS.
* The author has initiated a paper submission.
* The CMS file upload functionality is available.

---

## Main Success Scenario
1. The author is completing the paper submission form.
2. The author selects the option to upload a manuscript file.
3. The CMS prompts the author to choose a file from their local system.
4. The author selects a manuscript file and confirms the upload.
5. The CMS validates the file format.
6. The CMS validates the file size.
7. The CMS stores the manuscript file in the file storage system.
8. The CMS associates the uploaded file with the corresponding paper submission.
9. The CMS confirms that the manuscript file was uploaded successfully.

---

## Extensions

* **5a**: File format is not supported  
    * **5a1**: The CMS rejects the file and displays an error message indicating the allowed formats (PDF, Word, LaTeX).

* **6a**: File exceeds the maximum allowed size  
    * **6a1**: The CMS rejects the file and displays an error message indicating that the file size limit has been exceeded.

* **7a**: File storage system is unavailable  
    * **7a1**: The CMS displays an error message indicating that the file upload failed and the file was not saved.

* **4a**: Author cancels the file upload  
    * **4a1**: The CMS aborts the upload process and returns the author to the submission form without storing the file.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent during active paper submission periods  
* **Open Issues**:  
  * Whether partial or interrupted uploads should be resumable  
  * Whether uploaded files should be automatically scanned for viruses
