# Acceptance Test Suite — UC-22 Edit Conference Schedule

## Overview
**Use Case:** UC-22 Edit Conference Schedule  
**Primary Actor:** Editor  
**Objective:** Verify that an authenticated editor can modify an existing conference schedule, that all defined constraints are validated before saving, that valid edits are persisted, and that error and cancellation paths behave correctly.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable).
- A conference schedule already exists in the CMS.
- Schedule constraints include (at minimum):
  - No overlapping sessions in the same room
  - Valid room assignments
  - Valid session times
- Only editors (and possibly administrators) can edit schedules.
- Schedule edits are not published automatically unless explicitly saved.

### Seed Accounts / Entities
**E-01 Editor**
- Email/Username: `editor_user@example.com`
- Password: `ValidPass!234`

**A-01 Administrator**
- Email/Username: `admin_user@example.com`
- Password: `ValidPass!234`

### Schedule Data
**S-01 Existing Schedule**
- Schedule ID: `SCHED-1001`
- Status: Draft or Generated
- Sessions:
  - Session A: Room 101, 09:00–10:00
  - Session B: Room 102, 09:00–10:00

**S-02 Non-existent Schedule**
- Schedule ID: `SCHED-9999`
- Status: Does not exist

---

## UC-22-AT-01 — Editor can access schedule editing interface (Main Success Path)
**Priority:** P0  
**Preconditions:** Editor authenticated; S-01 exists.  
**Steps:**
1. Log in as editor (E-01).
2. Navigate to the conference schedule management area.
3. Select schedule S-01.
**Expected Results:**
- CMS displays S-01 in an editable format.
- All sessions, times, and room assignments are visible.
- Editing controls are enabled.

---

## UC-22-AT-02 — Editor successfully updates schedule with valid changes
**Priority:** P0  
**Test Data:** E-01 + S-01  
**Preconditions:** S-01 has no conflicts initially.  
**Steps:**
1. Open S-01 for editing.
2. Modify a session time (e.g., Session A from 09:00–10:00 to 10:00–11:00).
3. Submit the updated schedule.
**Expected Results:**
- CMS validates schedule constraints successfully.
- CMS saves the updated schedule to the database.
- CMS confirms that the schedule was updated successfully.
- Changes persist on reload.

---

## UC-22-AT-03 — Constraint violation prevents schedule save (Extension 6a)
**Priority:** P0  
**Test Data:** E-01 + S-01  
**Preconditions:** None.  
**Steps:**
1. Open S-01 for editing.
2. Modify Session A to overlap with Session B in the same room.
3. Submit the updated schedule.
**Expected Results:**
- CMS detects the constraint violation (overlap).
- CMS displays validation errors identifying the conflict.
- CMS prevents saving the schedule.
- Editor remains in the editing interface with changes unsaved.

---

## UC-22-AT-04 — Editor can correct violations and save successfully
**Priority:** P1  
**Preconditions:** UC-22-AT-03 has occurred; editor is still in edit mode.  
**Steps:**
1. Resolve the conflicting session times.
2. Resubmit the updated schedule.
**Expected Results:**
- CMS revalidates constraints.
- CMS saves the corrected schedule.
- CMS confirms successful update.

---

## UC-22-AT-05 — Editor cancels edit operation (Extension 7a)
**Priority:** P1  
**Test Data:** E-01 + S-01  
**Steps:**
1. Open S-01 for editing.
2. Make unsaved changes.
3. Select “Cancel” or navigate away without saving.
**Expected Results:**
- CMS discards unsaved changes.
- CMS restores the previous version of the schedule.
- No changes are persisted.

---

## UC-22-AT-06 — Database unavailable prevents saving edits (Extension 8a)
**Priority:** P0  
**Test Data:** E-01 + S-01  
**Preconditions:** Simulate CMS database outage.  
**Steps:**
1. Open S-01 for editing.
2. Make valid changes.
3. Attempt to save the schedule.
**Expected Results:**
- CMS displays an error indicating schedule changes could not be saved.
- No partial or inconsistent updates occur.
- Editor remains in edit mode or is prompted appropriately.

---

## UC-22-AT-07 — Editing a non-existent schedule is handled safely (Extension 3a)
**Priority:** P0  
**Test Data:** E-01 + S-02  
**Steps:**
1. Attempt to access S-02 directly via URL or stale UI link.
**Expected Results:**
- CMS displays an error indicating the schedule does not exist.
- CMS returns the editor to the schedule list.
- No editing interface is shown.

---

## UC-22-AT-08 — Web server unavailable behavior (Extension 1a)
**Priority:** P0  
**Preconditions:** Simulate CMS web server outage.  
**Steps:**
1. Attempt to access or save schedule edits.
**Expected Results:**
- CMS displays a system error page.
- No edits are saved.

---

## UC-22-AT-09 — Security: non-editor cannot edit schedules
**Priority:** P0  
**Preconditions:** Non-editor account exists (e.g., referee or author).  
**Steps:**
1. Log in as non-editor user.
2. Attempt to access schedule editing interface.
**Expected Results:**
- CMS denies access (403 or safe redirect).
- Editing controls are not visible.

---

## UC-22-AT-10 — Administrator access consistency (if applicable)
**Priority:** P2  
**Preconditions:** Administrator role is allowed to edit schedules.  
**Steps:**
1. Log in as administrator (A-01).
2. Access S-01 editing interface.
**Expected Results:**
- Behavior matches editor permissions.
- Constraint validation and save behavior is identical.
