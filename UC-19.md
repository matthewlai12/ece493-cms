# Use Case UC-19: Ensure Three Reviewers per Paper

**Goal in Context**:  
Ensure that each submitted paper is reviewed by exactly three referees so that the peer review process is fair, balanced, and consistent.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Editor

**Secondary Actors**:  
Referee  
CMS Web Interface  
CMS Database

**Trigger**:  
The editor assigns referees to a paper or a referee accepts a review invitation.

---

## Success End Condition
* The paper has exactly three assigned referees, and the system allows the review process to proceed.

## Failed End Condition
* The paper has fewer than or more than three referees assigned, preventing fair and consistent review.

---

## Preconditions
* The editor is registered and authenticated in the CMS.
* At least one paper exists that requires reviewer assignment.
* The CMS maintains an up-to-date list of referee assignments per paper.

---

## Main Success Scenario
1. The editor selects a submitted paper.
2. The CMS displays the list of currently assigned referees for the paper.
3. The editor assigns referees to the paper.
4. The CMS counts the total number of referees assigned to the paper.
5. The CMS determines that the paper has exactly three assigned referees.
6. The CMS confirms that the reviewer requirement has been satisfied and allows the review process to proceed.

---

## Extensions

* **4a**: Fewer than three referees are assigned  
    * **4a1**: The CMS informs the editor that additional referees must be assigned before reviews can begin.

* **4b**: More than three referees are assigned  
    * **4b1**: The CMS blocks the additional assignment and informs the editor that the maximum number of referees has been reached.

* **3a**: Assigned referee exceeds workload limit  
    * **3a1**: The CMS prevents assignment of the referee and informs the editor of the workload constraint.

* **2a**: CMS database is unavailable  
    * **2a1**: The CMS displays an error message indicating that referee assignments cannot be verified at this time.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent during referee assignment phase  
* **Open Issues**:  
  * Whether editors can override the three-reviewer constraint in exceptional cases  
  * Whether the system should automatically suggest additional referees when fewer than three are assigned
