# Use Case UC-07: Change Password

**Goal in Context**:  
Allow a registered user to securely change their account password in order to maintain or restore account security.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Registered User

**Secondary Actors**:  
CMS Web Interface  
CMS Database  
Authentication Service

**Trigger**:  
The user selects the “Change Password” option from their account or profile settings.

---

## Success End Condition
* The user’s password is successfully updated in the CMS database, and the new password is required for future logins.

## Failed End Condition
* The user’s password is not changed, and the original password remains valid.

---

## Preconditions
* The user is registered in the CMS.
* The user is authenticated and logged into the CMS.
* The CMS password management functionality is available.

---

## Main Success Scenario
1. The user is logged into the CMS.
2. The user navigates to account or security settings.
3. The CMS displays the change password form.
4. The user enters their current password.
5. The user enters a new password.
6. The user confirms the new password.
7. The user submits the password change request.
8. The CMS validates the current password.
9. The CMS validates the new password against security requirements.
10. The CMS updates the password in the database.
