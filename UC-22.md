# Use Case UC-22: Edit Conference Schedule

**Goal in Context**:  
Allow an editor to modify the conference schedule to resolve conflicts, satisfy constraints, and finalize session details before publication.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Editor

**Secondary Actors**:  
Administrator  
CMS Web Interface  
CMS Database

**Trigger**:  
The editor selects an existing conference schedule and initiates an edit action.

---

## Success End Condition
* The conference schedule is successfully updated and stored, with conflicts resolved and constraints satisfied.

## Failed End Condition
* The conference schedule is not updated, and unresolved conflicts remain.

---

## Preconditions
* The editor is registered and authenticated in the CMS.
* A conference schedule exists (generated or previously edited).
* The CMS schedule editing functionality is available.

---

## Main Success Scenario
1. The editor logs into the CMS.
2. The editor navigates to the conference schedule management area.
3. The editor selects an existing schedule to edit.
4. The CMS displays the schedule in an editable format.
5. The editor modifies schedule entries (e.g., session times, rooms, paper assignments).
6. The CMS validates the updated schedule against defined constraints (e.g., no overlaps, room availability).
7. The editor submits the updated schedule.
8. The CMS saves the updated schedule to the database.
9. The CMS confirms that the schedule has been successfully updated.

---

## Extensions

* **6a**: Updated schedule violates constraints  
    * **6a1**: The CMS displays validation errors identifying conflicts or violations.
    * **6a2**: The CMS prevents saving and returns the editor to the editing interface.

* **7a**: Editor cancels the edit operation  
    * **7a1**: The CMS discards unsaved changes and restores the previous schedule.

* **8a**: CMS database is unavailable  
    * **8a1**: The CMS displays an error message indicating that schedule changes could not be saved.

* **3a**: Selected schedule does not exist  
    * **3a1**: The CMS displays an error message and returns the editor to the schedule list.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: Medium–High  
* **Frequency**: Occasional, during schedule finalization  
* **Open Issues**:  
  * Whether editors can edit schedules after publication  
  * Whether version history of schedule edits should be maintained
