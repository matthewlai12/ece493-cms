# Use Case UC-26: Register for Conference

**Goal in Context**:  
Allow an attendee to register for the conference so they can participate in conference activities.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Attendee

**Secondary Actors**:  
CMS Web Interface  
CMS Database  
Payment System (external)

**Trigger**:  
The attendee selects the option to register for the conference.

---

## Success End Condition
* The attendee is successfully registered for the conference and proceeds to payment or confirmation.

## Failed End Condition
* The attendee is not registered for the conference.

---

## Preconditions
* The attendee is registered and authenticated in the CMS.
* Conference registration is open.
* Conference pricing information is available.
* The CMS registration functionality is available.

---

## Main Success Scenario
1. The attendee logs into the CMS.
2. The attendee navigates to the conference registration page.
3. The CMS displays registration details and pricing information.
4. The attendee selects the option to register for the conference.
5. The CMS records the attendee’s intent to register.
6. The CMS directs the attendee to the payment process.
7. The CMS confirms that the attendee has been successfully registered pending payment completion.

---

## Extensions

* **2a**: Attendee is not authenticated  
    * **2a1**: The CMS prompts the attendee to log in or create an account before registering.

* **3a**: Registration is closed  
    * **3a1**: The CMS displays a message indicating that conference registration is no longer available.

* **4a**: Attendee cancels registration  
    * **4a1**: The CMS aborts the registration process and returns the attendee to the dashboard.

* **6a**: Payment system is unavailable  
    * **6a1**: The CMS displays an error message and pauses the registration process.

* **5a**: CMS database is unavailable  
    * **5a1**: The CMS displays an error message indicating that registration cannot be completed.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Once per attendee  
* **Open Issues**:  
  * Whether attendees can edit registration details before payment  
  * Whether registration capacity limits should be enforced
