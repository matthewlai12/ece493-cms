# Use Case UC-15: Access Assigned Papers

**Goal in Context**:  
Allow a referee to view and open all papers that have been assigned to them for review so they can perform the review tasks.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Referee

**Secondary Actors**:  
CMS Web Interface  
CMS Database  
File Storage System

**Trigger**:  
The referee selects the option to view assigned papers after logging into the CMS.

---

## Success End Condition
* The referee successfully views the list of assigned papers and can access the manuscript files for review.

## Failed End Condition
* The referee cannot view or access the papers assigned to them.

---

## Preconditions
* The referee is registered in the CMS.
* The referee is authenticated and logged into the CMS.
* The referee has accepted at least one review invitation.
* The CMS paper access functionality is available.

---

## Main Success Scenario
1. The referee logs into the CMS.
2. The referee navigates to their dashboard.
3. The referee selects the option to view assigned papers.
4. The CMS retrieves the list of papers assigned to the referee from the database.
5. The CMS displays the list of assigned papers.
6. The referee selects a paper from the list.
7. The CMS retrieves the manuscript file from the file storage system.
8. The CMS displays or downloads the manuscript file for the referee.

---

## Extensions

* **4a**: No papers are currently assigned to the referee  
    * **4a1**: The CMS displays a message indicating that no papers are assigned.

* **7a**: Manuscript file is unavailable or corrupted  
    * **7a1**: The CMS displays an error message indicating the paper cannot be accessed at this time.

* **4b**: CMS database is unavailable  
    * **4b1**: The CMS displays an error message indicating assigned papers cannot be retrieved.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

* **6a**: Referee cancels paper access  
    * **6a1**: The CMS returns the referee to the list of assigned papers without opening the manuscript.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent during the paper review period  
* **Open Issues**:  
  * Whether referees should be able to download papers or only view them online  
  * Whether access should be revoked after review deadlines
