# Acceptance Test Suite — UC-26 Register for Conference

## Overview
**Use Case:** UC-26 Register for Conference  
**Primary Actor:** Attendee  
**Objective:** Verify that an authenticated attendee can register for the conference when registration is open, that the system correctly handles redirection to payment, and that all failure and exception paths are handled safely.

---

## Assumptions / Test Data
- Supported browsers: Chrome, Firefox (latest stable)
- Conference registration window is configurable (open/closed)
- Payment processing is handled by an external system
- Only authenticated attendees may register

### Seed Accounts
**ATT-01 Attendee**  
- Email: `attendee_user@example.com`  
- Password: `ValidPass!123`

### Conference State
- Registration status: **Open**
- Pricing information: Available
- Registration capacity: Not exceeded

---

## UC-26-AT-01 — Attendee can access conference registration page (Main Success Path)
**Priority:** P0  
**Preconditions:** Attendee authenticated; registration open.  
**Steps:**
1. Log in as attendee (ATT-01).
2. Navigate to the CMS dashboard.
3. Select the “Conference Registration” option.
**Expected Results:**
- CMS displays conference registration page.
- Pricing and registration details are visible.

---

## UC-26-AT-02 — Successful conference registration initiation (Main Success Path)
**Priority:** P0  
**Preconditions:** Registration page accessible.  
**Steps:**
1. Select “Register for Conference.”
2. Confirm intent to register.
**Expected Results:**
- CMS records attendee’s intent to register.
- CMS updates registration status to “Pending Payment.”
- Attendee is redirected to the payment process.

---

## UC-26-AT-03 — Registration pending payment confirmation (Main Success Path)
**Priority:** P0  
**Preconditions:** Registration initiated.  
**Steps:**
1. Complete steps up to payment redirection.
**Expected Results:**
- CMS confirms attendee is registered pending payment.
- Attendee cannot re-register while status is pending.

---

## UC-26-AT-04 — Attendee not authenticated (Extension 2a)
**Priority:** P0  
**Preconditions:** Attendee not logged in.  
**Steps:**
1. Attempt to access the conference registration page.
**Expected Results:**
- CMS prompts the user to log in or create an account.
- Registration process does not proceed.

---

## UC-26-AT-05 — Registration closed (Extension 3a)
**Priority:** P0  
**Preconditions:** Conference registration status set to “Closed.”  
**Steps:**
1. Log in as attendee.
2. Attempt to access the registration page.
**Expected Results:**
- CMS displays message indicating registration is closed.
- Registration option is disabled or unavailable.

---

## UC-26-AT-06 — Attendee cancels registration (Extension 4a)
**Priority:** P1  
**Preconditions:** Registration page accessible.  
**Steps:**
1. Select “Register for Conference.”
2. Cancel the registration action.
**Expected Results:**
- CMS aborts the registration process.
- No registration record is created.
- Attendee is returned to dashboard.

---

## UC-26-AT-07 — Payment system unavailable (Extension 6a)
**Priority:** P0  
**Preconditions:** Simulate payment system outage.  
**Steps:**
1. Attempt to register for the conference.
2. Reach payment redirection step.
**Expected Results:**
- CMS displays error indicating payment system is unavailable.
- Registration is paused and not finalized.
- Attendee is informed that they may retry later.

---

## UC-26-AT-08 — CMS database unavailable during registration (Extension 5a)
**Priority:** P0  
**Preconditions:** Simulate CMS database outage.  
**Steps:**
1. Attempt to register for the conference.
**Expected Results:**
- CMS displays error indicating registration cannot be completed.
- No registration data is stored.
- Attendee remains unregistered.

---

## UC-26-AT-09 — CMS web server unavailable (Extension 1a)
**Priority:** P0  
**Preconditions:** CMS web server unavailable.  
**Steps:**
1. Attempt to access the CMS registration page.
**Expected Results:**
- CMS displays a system error page.
- No registration action is performed.

---

## UC-26-AT-10 — Registration persistence after navigation
**Priority:** P1  
**Preconditions:** Registration initiated successfully.  
**Steps:**
1. Navigate away from the registration page.
2. Return to the dashboard.
**Expected Results:**
- CMS reflects attendee’s registration status correctly (e.g., pending payment).
- Duplicate registration is prevented.
