# Use Case UC-13: Receive Review Invitation

**Goal in Context**:  
Notify a referee that they have been invited to review a submitted paper so they can decide whether to accept or decline the review assignment.

**Scope**:  
Conference Management System (CMS)

**Level**:  
User Goal

**Primary Actor**:  
Referee

**Secondary Actors**:  
Editor  
CMS Web Interface  
CMS Database  
Notification Service (e.g., email)

**Trigger**:  
The editor assigns a referee to a submitted paper.

---

## Success End Condition
* The referee successfully receives a review invitation and can view the paper details and respond to the invitation.

## Failed End Condition
* The referee does not receive or cannot view the review invitation.

---

## Preconditions
* The referee is registered in the CMS.
* The editor has initiated referee assignment for a submitted paper.
* The referee has not exceeded the maximum allowed number of assigned papers.
* The CMS notification mechanism is available.

---

## Main Success Scenario
1. The editor selects a submitted paper for reviewer assignment.
2. The editor enters the referee’s email address.
3. The CMS verifies that the referee exists and is eligible to review additional papers.
4. The CMS records a pending review invitation for the referee.
5. The CMS sends a review invitation notification to the referee.
6. The referee receives the invitation and accesses it through the CMS or the notification link.

---

## Extensions

* **3a**: Referee has reached the maximum allowed number of review assignments  
    * **3a1**: The CMS blocks the assignment and informs the editor that the referee cannot be assigned.

* **3b**: Referee email address does not correspond to a registered referee  
    * **3b1**: The CMS informs the editor that the referee is not registered and does not send an invitation.

* **5a**: Notification delivery fails  
    * **5a1**: The CMS logs the failure and allows the referee to view the invitation upon logging into the CMS.

* **4a**: CMS database is unavailable  
    * **4a1**: The CMS displays an error message and does not record or send the invitation.

* **1a**: CMS web server is unavailable  
    * **1a1**: The CMS displays a system error page indicating temporary service unavailability.

---

## Related Information
* **Priority**: High  
* **Frequency**: Frequent during the paper review assignment phase  
* **Open Issues**:  
  * Whether review invitations should expire after a fixed period  
  * Whether referees should see limited paper metadata before accepting the invitation
